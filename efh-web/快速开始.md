## 🚀 快速开始

### 方式一：本地开发（推荐先测试）

```bash
# 1. 进入前端目录
cd efh-web

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run dev
```

**访问：** http://localhost:3000

**注意：** 需要先启动后端服务（网关、用户服务、社区服务）

---

### 方式二：Docker 部署（生产环境）

#### 第一步：构建前端镜像

```bash
# 在 efh-web 目录
cd efh-web
docker build -t efh-web:latest .
```

#### 第二步：运行前端容器

```bash
docker run -d \
  --name efh-web \
  --network ev-forklift-hub_efh-network \
  -p 80:80 \
  efh-web:latest
```

#### 第三步：访问应用

```
前端：http://localhost
```

---

### 方式三：使用 docker-compose（最简单）

**需要先更新 docker-compose.yml 添加前端服务！**

在 `docker-compose.yml` 的 `services` 部分添加：

```yaml
  # 前端 Web 应用
  web:
    build:
      context: ./efh-web
      dockerfile: Dockerfile
    container_name: efh-web
    restart: always
    ports:
      - "80:80"
    networks:
      - efh-network
    depends_on:
      - gateway
```

然后启动：

```bash
# 在项目根目录
docker-compose up -d web
```

## 🌐 API 代理说明

### 开发环境（vite.config.js）

```javascript
proxy: {
  '/api': {
    target: 'http://localhost:8080',  // 后端网关
    changeOrigin: true
  }
}
```

**请求流程：**
```
前端 (localhost:3000) 
  → Vite 代理 
  → 后端网关 (localhost:8080)
```

### 生产环境（nginx.conf）

```nginx
location /api/ {
    proxy_pass http://gateway:8080/;
}
```

**请求流程：**
```
浏览器 
  → Nginx (localhost:80) 
  → 后端网关 (gateway:8080)
```

## 🐛 常见问题

### Q: npm install 失败？

**A:** 尝试使用国内镜像：

```bash
npm config set registry https://registry.npmmirror.com
npm install
```

### Q: 开发服务器启动失败？

**A:** 检查端口 3000 是否被占用：

```bash
# Windows
netstat -ano | findstr :3000

# Linux/Mac
lsof -i :3000
```

### Q: Docker 构建很慢？

**A:** 使用国内镜像加速：

```dockerfile
# 在 Dockerfile 中添加
RUN npm config set registry https://registry.npmmirror.com
```

### Q: 前端无法调用后端 API？

**A:** 检查：
1. 后端服务是否启动
2. 网关端口是否正确（8080）
3. 网络是否连通
