# 移动端访问指南

> 完整 App 安装说明见 **[手机端App使用指南.md](./手机端App使用指南.md)**（添加到主屏幕、PWA、Capacitor 打包等）。

前端已适配手机浏览器（宽度 ≤ 768px 时自动切换移动端布局：底部导航栏、侧滑菜单、响应式栅格），并采用简约大气的统一视觉风格（`src/styles/theme.css`）。

## 方式一：局域网访问（开发/内网演示）

适合同事在同一 WiFi 下用手机访问你电脑上的开发环境。

### 1. 启动后端

确保网关 `8080` 及各微服务已启动（MySQL、Redis、Nacos 等基础设施也需运行）。

### 2. 启动前端

```bash
cd efh-web
npm install
npm run dev
```

`vite.config.js` 已配置 `host: '0.0.0.0'`，会监听所有网卡。

### 3. 查本机 IP

**Windows（PowerShell）：**

```powershell
ipconfig
# 找到 WLAN 或以太网的 IPv4，例如 192.168.1.100
```

**macOS / Linux：**

```bash
ip addr | grep inet
# 或 ifconfig
```

### 4. 手机浏览器访问

手机连接**同一 WiFi**，打开：

```
http://192.168.1.100:3000
```

将 IP 换成你的实际地址。Vite 会把 `/user`、`/community` 等 API 代理到本机 `8080`，无需改代码。

### 常见问题

| 现象 | 处理 |
|------|------|
| 手机打不开页面 | 检查电脑防火墙是否放行 3000 端口 |
| 页面能开但接口失败 | 确认网关 8080 已启动 |
| 公司 WiFi 设备隔离 | 改用手机热点，或部署到服务器（方式二） |

**Windows 防火墙放行（管理员 PowerShell）：**

```powershell
New-NetFirewallRule -DisplayName "EFH Web Dev" -Direction Inbound -LocalPort 3000 -Protocol TCP -Action Allow
```

## 方式二：服务器部署（推荐生产/外网）

将项目部署到云服务器后，同事通过公网 IP 或域名访问。

```bash
# 构建前端
cd efh-web && npm run build

# Docker 启动（含 Nginx 前端 + API 反代）
docker compose up -d web gateway ...
```

访问：`http://你的服务器IP/` 或 `https://你的域名/`

Nginx 会把 API 请求转发到网关，手机与 PC 使用同一地址即可。

## 移动端界面说明

- **底部 Tab**：首页、商城、知识库、AI、我的
- **右上角菜单按钮**：侧滑打开完整导航（维修服务、订单、收藏、发帖等）
- **购物车**：顶部购物车图标
- 表格类页面（订单、购物车）支持横向滑动查看
