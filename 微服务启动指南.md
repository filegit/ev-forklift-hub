# 微服务架构启动指南

## 架构说明

本项目已改造为 **Spring Boot + Spring Cloud 微服务架构**，包含以下技术栈：

- **服务注册与发现**: Nacos
- **配置中心**: Nacos Config
- **API网关**: Spring Cloud Gateway
- **服务调用**: OpenFeign + LoadBalancer
- **数据库**: MySQL + MyBatis-Plus
- **分库分表**: ShardingSphere
- **缓存**: Redis + Redisson（分布式锁）
- **消息队列**: Kafka
- **搜索引擎**: Elasticsearch
- **并发编程**: 线程池、异步任务、CompletableFuture
- **JVM优化**: G1GC、堆内存调优、GC日志

## 环境准备

### 1. 安装Nacos（服务注册中心）

```bash
# 下载Nacos 2.2.3
wget https://github.com/alibaba/nacos/releases/download/2.2.3/nacos-server-2.2.3.zip
unzip nacos-server-2.2.3.zip
cd nacos/bin

# Windows启动
startup.cmd -m standalone

# Linux/Mac启动
sh startup.sh -m standalone

# 访问控制台: http://localhost:8848/nacos
# 默认账号密码: nacos/nacos
```

### 2. 安装MySQL（数据库）

创建数据库：
```sql
-- 用户服务数据库（分库）
CREATE DATABASE efh_user_0 DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE efh_user_1 DEFAULT CHARACTER SET utf8mb4;

-- 社区服务数据库（分库）
CREATE DATABASE efh_community_0 DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE efh_community_1 DEFAULT CHARACTER SET utf8mb4;

-- 配件服务数据库
CREATE DATABASE efh_parts DEFAULT CHARACTER SET utf8mb4;

-- 维修服务数据库
CREATE DATABASE efh_service DEFAULT CHARACTER SET utf8mb4;
```

### 3. 安装Redis（缓存）

```bash
# Windows: 下载Redis安装包
# Linux/Mac
brew install redis  # Mac
sudo apt install redis  # Ubuntu

# 启动Redis
redis-server
```

### 4. 安装Kafka（消息队列）

```bash
# 下载Kafka
wget https://downloads.apache.org/kafka/3.5.1/kafka_2.13-3.5.1.tgz
tar -xzf kafka_2.13-3.5.1.tgz
cd kafka_2.13-3.5.1

# 启动Zookeeper
bin/zookeeper-server-start.sh config/zookeeper.properties

# 启动Kafka
bin/kafka-server-start.sh config/server.properties
```

### 5. 安装Elasticsearch（搜索引擎）

```bash
# 下载ES 7.17.9
wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-7.17.9.tar.gz
tar -xzf elasticsearch-7.17.9.tar.gz
cd elasticsearch-7.17.9

# 启动ES
bin/elasticsearch

# 访问: http://localhost:9200
```

## 启动步骤

### 1. 编译打包

```bash
cd ev-forklift-hub
mvn clean package -DskipTests
```

### 2. 启动顺序

**必须按以下顺序启动：**

#### 第一步：启动基础设施
1. Nacos
2. MySQL
3. Redis
4. Kafka
5. Elasticsearch

#### 第二步：启动微服务

```bash
# 1. 启动网关服务
cd efh-gateway/target
java -jar efh-gateway-1.0.0.jar

# 2. 启动用户服务
cd efh-user/target
java -jar efh-user-1.0.0.jar

# 3. 启动社区服务
cd efh-community/target
java -jar efh-community-1.0.0.jar

# 4. 启动配件服务
cd efh-parts/target
java -jar efh-parts-1.0.0.jar

# 5. 启动维修服务
cd efh-service/target
java -jar efh-service-1.0.0.jar
```

### 3. 验证服务

访问Nacos控制台查看服务注册情况：
- URL: http://localhost:8848/nacos
- 账号/密码: nacos/nacos

所有服务应该显示为 **UP** 状态。

## 核心功能说明

### 1. 分库分表（ShardingSphere）

- **用户表**: 按用户ID分2个库，每个库2张表
- **帖子表**: 按用户ID分2个库，每个库4张表
- **订单表**: 按订单ID分4张表

### 2. Redis缓存

- 用户信息缓存（30分钟过期）
- 热点数据缓存
- 分布式锁（Redisson）

### 3. Kafka消息队列

- 用户注册消息: `user-register-topic`
- 用户登录日志: `user-login-topic`
- 帖子创建消息: `post-create-topic`
- 帖子点赞消息: `post-like-topic`

### 4. Elasticsearch搜索

- 帖子全文搜索
- 支持中文分词（ik_max_word）

### 5. 并发编程

- **异步任务线程池**: 处理异步业务
- **IO密集型线程池**: 处理IO操作
- **CPU密集型线程池**: 处理计算任务
- **CompletableFuture**: 异步编排

### 6. JVM优化

- 使用G1垃圾收集器
- 堆内存调优
- GC日志记录
- OOM自动dump

详见：`JVM配置说明.md`

## 常见问题

### 1. Nacos连接失败
- 检查Nacos是否启动
- 确认端口8848未被占用

### 2. 数据库连接失败
- 检查MySQL是否启动
- 确认数据库已创建
- 检查用户名密码是否正确

### 3. Redis连接失败
- 检查Redis是否启动
- 确认端口6379未被占用

### 4. 服务启动失败
- 查看日志文件
- 检查端口是否被占用
- 确认依赖服务已启动

## 开发建议

1. **本地开发**: 可以只启动需要的服务
2. **日志级别**: 开发环境使用debug，生产环境使用info
3. **性能监控**: 使用Arthas、Skywalking等工具
4. **压力测试**: 使用JMeter进行压测
