## 🔧 配置说明

### 1. 数据库配置

每个服务的 `application.yml` 中都有数据库配置：

```yaml
spring:
  shardingsphere:
    datasource:
      ds0:
        jdbc-url: jdbc:mysql://localhost:3306/数据库名
        username: root      # 修改为你的用户名
        password: 123456    # 修改为你的密码
```

### 2. Nacos 配置

每个服务的 `bootstrap.yml` 中都有 Nacos 配置：

```yaml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: 127.0.0.1:8848  # Nacos 地址
        namespace: dev                # 命名空间
      config:
        server-addr: 127.0.0.1:8848  # Nacos 地址
        namespace: dev                # 命名空间
```

### 3. Redis 配置

```yaml
spring:
  redis:
    host: 127.0.0.1  # Redis 地址
    port: 6379       # Redis 端口
    database: 0      # 数据库索引（不同服务使用不同索引）
```

## 🐛 常见问题解决

### 1. Nacos 连接失败
```
错误信息: Connection refused: connect
解决方案:
1. 确认 Nacos 已启动：访问 http://localhost:8848/nacos
2. 检查 bootstrap.yml 中的 Nacos 地址
3. 检查防火墙是否阻止 8848 端口
```

### 2. 数据库连接失败
```
错误信息: Communications link failure
解决方案:
1. 确认 MySQL 服务已启动
2. 检查数据库名、用户名、密码是否正确
3. 确认数据库已创建：SHOW DATABASES;
4. 检查 MySQL 端口是否为 3306
```

### 3. Redis 连接失败
```
错误信息: Unable to connect to Redis
解决方案:
1. 确认 Redis 服务已启动：redis-cli ping
2. 检查 Redis 地址和端口
3. 检查 Redis 是否设置了密码
```

### 4. 端口被占用
```
错误信息: Port 8080 was already in use
解决方案:
# Windows 查看端口占用
netstat -ano | findstr :8080

# 结束占用端口的进程
taskkill /PID 进程ID /F

# 或修改 application.yml 中的端口号
```

### 5. 依赖下载失败
```
错误信息: Could not resolve dependencies
解决方案:
1. 配置国内 Maven 镜像（阿里云）
2. 删除本地仓库中的损坏文件
3. 重新执行：mvn clean install -U
```

### 6. 内存不足
```
错误信息: OutOfMemoryError
解决方案:
# 增加 JVM 堆内存
java -Xms512m -Xmx1024m -jar xxx.jar

# 或在 IDEA 中设置 VM options:
-Xms512m -Xmx1024m
```
