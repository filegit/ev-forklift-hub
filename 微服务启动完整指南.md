## 🔧 如果某个服务启动失败

### 常见问题和解决方案

#### 1. 数据库连接失败

**错误信息：** `Communications link failure`

**解决方案：**
```bash
# 检查 MySQL 是否运行
# Windows
net start mysql

# 检查数据库是否存在
mysql -uroot -p123456 -e "SHOW DATABASES;"

# 如果数据库不存在，执行 database.sql
mysql -uroot -p123456 < database.sql
```

#### 2. Redis 连接失败

**错误信息：** `Unable to connect to Redis`

**解决方案：**
```bash
# 启动 Redis
redis-server

# 或使用 Docker
docker run -d -p 6379:6379 redis:7-alpine redis-server --requirepass 123456
```

#### 3. Kafka 连接失败（可选）

**错误信息：** `Connection to node -1 could not be established`

**解决方案：**

如果不需要 Kafka 功能，可以注释掉 Kafka 配置：

```yaml
# spring:
#   kafka:
#     bootstrap-servers: 127.0.0.1:9092
```

或者启动 Kafka：
```bash
docker run -d -p 9092:9092 --name kafka \
  -e KAFKA_ZOOKEEPER_CONNECT=zookeeper:2181 \
  confluentinc/cp-kafka:7.5.0
```

#### 4. Elasticsearch 连接失败（可选）

**错误信息：** `Connection refused`

**解决方案：**

Elasticsearch 已经在 Docker 中运行，检查是否可访问：
```bash
curl http://localhost:9200
```

如果不需要搜索功能，可以注释掉 ES 配置。

## 💡 启动技巧

### 1. 查看启动日志

在 IDEA 控制台查看日志，看到以下信息表示成功：
```
========== XXX服务启动成功 ==========
```

### 2. 如果启动慢

第一次启动可能较慢，因为需要：
- 连接 Nacos
- 初始化数据库连接池
- 加载配置

耐心等待 30-60 秒。

### 3. 并行启动

可以同时启动多个服务，不需要等待前一个完全启动。
