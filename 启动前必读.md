# ⚠️ 重要：启动前必读

## 问题已修复 ✅

已为所有微服务添加了 `spring-cloud-starter-bootstrap` 依赖，这是 Spring Cloud 2021.x 版本加载 bootstrap.yml 的必需依赖。

## 🔧 立即执行以下步骤

### 第一步：刷新 Maven 依赖（必须！）

**在 IDEA 中：**
1. 点击右侧的 Maven 工具栏
2. 点击刷新按钮（Reload All Maven Projects）
3. 或者右键项目根目录 → Maven → Reload Project

**或者在命令行执行：**
```bash
mvn clean install -DskipTests
```

### 第二步：确保基础设施已启动

- ✅ MySQL (端口 3306)
- ✅ Redis (端口 6379)  
- ✅ Nacos (端口 8848)

**启动 Nacos：**
```bash
# Windows
cd nacos/bin
startup.cmd -m standalone

# 访问控制台
http://localhost:8848/nacos
账号密码: nacos/nacos
```

### 第三步：初始化数据库

```sql
-- 创建数据库
CREATE DATABASE efh_user_0 DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE efh_user_1 DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE efh_community_0 DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE efh_community_1 DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE efh_parts DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE efh_service DEFAULT CHARACTER SET utf8mb4;

-- 执行建表脚本
source database.sql;
```

### 第四步：修改数据库密码

修改各服务 `application.yml` 中的密码：
```yaml
spring:
  shardingsphere:
    datasource:
      ds0:
        username: root
        password: 你的密码  # 修改这里
```

### 第五步：启动微服务

**按以下顺序启动：**

1. **efh-gateway** (GatewayApplication.java) - 端口 8080
2. **efh-user** (UserApplication.java) - 端口 8081
3. **efh-community** (CommunityApplication.java) - 端口 8082
4. **efh-parts** (PartsApplication.java) - 端口 8083
5. **efh-service** (ServiceApplication.java) - 端口 8084

**等待每个服务启动成功后再启动下一个！**

看到以下日志表示启动成功：
```
========== XXX服务启动成功 ==========
```

### 第六步：验证

访问 Nacos 控制台：http://localhost:8848/nacos

在"服务管理 > 服务列表"中应该看到 5 个服务：
- efh-gateway
- efh-user
- efh-community
- efh-parts
- efh-service

## 🐛 如果还有问题

### 1. 依赖下载失败
```bash
# 清理并重新下载
mvn clean install -U -DskipTests
```

### 2. Nacos 连接失败
- 确认 Nacos 已启动：访问 http://localhost:8848/nacos
- 检查 bootstrap.yml 中的 Nacos 地址是否正确

### 3. 数据库连接失败
- 确认 MySQL 服务已启动
- 确认数据库已创建
- 确认用户名密码正确

### 4. 端口被占用
```bash
# Windows 查看端口占用
netstat -ano | findstr :8080

# 结束进程
taskkill /PID 进程ID /F
```
