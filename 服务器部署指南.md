# 🚀 EV-Forklift-Hub 服务器部署指南

## 📋 部署前准备

### 1. 服务器要求

**最低配置：**
- CPU: 2核
- 内存: 4GB
- 硬盘: 40GB
- 操作系统: Ubuntu 20.04 / CentOS 7+

**推荐配置：**
- CPU: 4核
- 内存: 8GB
- 硬盘: 100GB
- 操作系统: Ubuntu 22.04

---

### 2. 需要安装的软件

- ✅ Docker (20.10+)
- ✅ Docker Compose (2.0+)
- ✅ Git
- ✅ JDK 8 (可选，如果不用 Docker 部署微服务)

---

## 🔧 服务器环境准备

### 第一步：安装 Docker

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Docker
curl -fsSL https://get.docker.com | bash

# 启动 Docker
sudo systemctl start docker
sudo systemctl enable docker

# 验证安装
docker --version
```

---

### 第二步：安装 Docker Compose

```bash
# 下载 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/download/v2.20.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

# 添加执行权限
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker-compose --version
```

---

### 第三步：配置防火墙

```bash
# 开放必要端口
sudo ufw allow 80/tcp      # HTTP
sudo ufw allow 443/tcp     # HTTPS
sudo ufw allow 8080/tcp    # 网关
sudo ufw allow 8848/tcp    # Nacos
sudo ufw allow 22/tcp      # SSH

# 启用防火墙
sudo ufw enable

# 查看状态
sudo ufw status
```

## 🚀 方式一：完整 Docker 部署

### 第一步：上传代码到服务器

```bash
# 在本地打包代码
cd d:\LH\ev-forklift-hub
git init
git add .
git commit -m "Initial commit"

# 推送到 Git 仓库（GitHub/Gitee）
git remote add origin YOUR_GIT_REPO_URL
git push -u origin master
```

**在服务器上：**

```bash
# 克隆代码
cd /opt
sudo git clone YOUR_GIT_REPO_URL ev-forklift-hub
cd ev-forklift-hub

# 赋予权限
sudo chmod +x docker-start.bat
```

---

### 第二步：修改生产环境配置

**创建生产环境配置文件：**

```bash
# 创建 .env 文件
cat > .env << EOF
# 数据库配置
MYSQL_ROOT_PASSWORD=YOUR_STRONG_PASSWORD
MYSQL_HOST=mysql
MYSQL_PORT=3306

# Redis 配置
REDIS_PASSWORD=YOUR_STRONG_PASSWORD
REDIS_HOST=redis
REDIS_PORT=6379

# Nacos 配置
NACOS_SERVER_ADDR=nacos:8848
NACOS_USERNAME=nacos
NACOS_PASSWORD=YOUR_STRONG_PASSWORD

# 应用环境
SPRING_PROFILES_ACTIVE=prod
EOF
```

---

### 第三步：构建并启动服务

```bash
# 1. 启动基础设施
docker-compose up -d mysql redis nacos elasticsearch

# 2. 等待基础设施启动（约 2 分钟）
sleep 120

# 3. 验证基础设施
docker-compose ps

# 4. 构建微服务镜像
docker-compose build

# 5. 启动所有微服务
docker-compose up -d

# 6. 查看所有服务状态
docker-compose ps

# 7. 查看日志
docker-compose logs -f
```

---

### 第四步：验证部署

```bash
# 验证网关
curl http://localhost:8080/actuator/health

# 验证用户服务
curl http://localhost:8081/api/test

# 验证社区服务
curl http://localhost:8082/api/post/test
```

## 🚀 方式二：混合部署

### 第一步：启动基础设施

```bash
cd /opt/ev-forklift-hub

# 只启动基础设施
docker-compose up -d mysql redis nacos elasticsearch

# 验证
docker-compose ps
```

---

### 第二步：打包微服务

**在本地开发机器上：**

```bash
cd d:\LH\ev-forklift-hub

# Maven 打包
mvn clean package -DskipTests

# 打包后的文件在：
# efh-gateway/target/efh-gateway-1.0.0.jar
# efh-user/target/efh-user-1.0.0.jar
# efh-community/target/efh-community-1.0.0.jar
# efh-parts/target/efh-parts-1.0.0.jar
# efh-service/target/efh-service-1.0.0.jar
```

---

### 第三步：上传 JAR 包到服务器

```bash
# 使用 scp 上传
scp efh-gateway/target/efh-gateway-1.0.0.jar user@server:/opt/efh/
scp efh-user/target/efh-user-1.0.0.jar user@server:/opt/efh/
scp efh-community/target/efh-community-1.0.0.jar user@server:/opt/efh/
scp efh-parts/target/efh-parts-1.0.0.jar user@server:/opt/efh/
scp efh-service/target/efh-service-1.0.0.jar user@server:/opt/efh/
```

---

### 第四步：创建启动脚本

**在服务器上：**

```bash
# 创建启动脚本
cat > /opt/efh/start-all.sh << 'EOF'
#!/bin/bash

# 启动用户服务
nohup java -jar -Xms512m -Xmx512m \
  -Dspring.profiles.active=prod \
  efh-user-1.0.0.jar > user.log 2>&1 &

# 等待 10 秒
sleep 10

# 启动社区服务
nohup java -jar -Xms512m -Xmx512m \
  -Dspring.profiles.active=prod \
  efh-community-1.0.0.jar > community.log 2>&1 &

# 等待 10 秒
sleep 10

# 启动配件服务
nohup java -jar -Xms512m -Xmx512m \
  -Dspring.profiles.active=prod \
  efh-parts-1.0.0.jar > parts.log 2>&1 &

# 等待 10 秒
sleep 10

# 启动维修服务
nohup java -jar -Xms512m -Xmx512m \
  -Dspring.profiles.active=prod \
  efh-service-1.0.0.jar > service.log 2>&1 &

# 等待 10 秒
sleep 10

# 启动网关服务
nohup java -jar -Xms512m -Xmx512m \
  -Dspring.profiles.active=prod \
  efh-gateway-1.0.0.jar > gateway.log 2>&1 &

echo "所有服务已启动！"
EOF

# 赋予执行权限
chmod +x /opt/efh/start-all.sh

# 启动所有服务
cd /opt/efh
./start-all.sh
```

---

### 第五步：创建停止脚本

```bash
cat > /opt/efh/stop-all.sh << 'EOF'
#!/bin/bash

# 停止所有 Java 进程
pkill -f "efh-.*\.jar"

echo "所有服务已停止！"
EOF

chmod +x /opt/efh/stop-all.sh
```

## 🔒 安全配置

### 1. 修改默认密码

```bash
# 修改 MySQL 密码
docker exec -it efh-mysql mysql -uroot -p
ALTER USER 'root'@'%' IDENTIFIED BY 'NEW_STRONG_PASSWORD';
FLUSH PRIVILEGES;

# 修改 Redis 密码
# 编辑 docker-compose.yml
# redis:
#   command: redis-server --requirepass NEW_STRONG_PASSWORD

# 修改 Nacos 密码
# 访问 http://server-ip:8848/nacos
# 登录后在用户管理中修改密码
```

---

### 2. 配置 HTTPS（使用 Nginx）

```bash
# 安装 Nginx
sudo apt install nginx -y

# 安装 Certbot（Let's Encrypt）
sudo apt install certbot python3-certbot-nginx -y

# 获取 SSL 证书
sudo certbot --nginx -d yourdomain.com

# Nginx 配置
sudo nano /etc/nginx/sites-available/efh

# 添加以下配置：
server {
    listen 80;
    server_name yourdomain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl;
    server_name yourdomain.com;

    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;

    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# 启用配置
sudo ln -s /etc/nginx/sites-available/efh /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```
