## 🔑 关键设计

### 1. 微服务认证流程

```
用户请求 
  → 网关 (验证 Token)
  → 提取 userId
  → 添加请求头 X-User-Id
  → 转发到微服务
  → 微服务从请求头获取 userId
  → 处理业务逻辑
```

**优势：**
- 统一认证
- 微服务无需解析 Token
- 简化微服务代码

### 2. JWT Token 设计

- 使用固定密钥（支持服务重启）
- Token 有效期：7天
- 包含用户信息：userId、username

### 3. 前后端分离

- 前端：Vue 3 SPA
- 后端：RESTful API
- 通过 Nginx 代理统一访问

## 🐛 已解决的问题

### 1. JWT 密钥问题
- **问题：** 使用动态密钥导致服务重启后 Token 失效
- **解决：** 使用固定密钥字符串

### 2. 微服务重复解析 Token
- **问题：** 微服务尝试解析 Token 导致错误
- **解决：** 网关统一认证，微服务从请求头获取用户信息

### 3. 数据库表结构问题
- **问题：** id 字段未设置自增，category 字段类型错误
- **解决：** 修改表结构，设置 id 自增，category 改为 INT

## 🌐 部署到服务器

### 1. 准备服务器

```bash
# 安装 Docker
curl -fsSL https://get.docker.com | sh

# 安装 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
```

### 2. 上传项目

```bash
# 使用 git
git clone https://github.com/your-repo/ev-forklift-hub.git

# 或使用 scp
scp -r ev-forklift-hub user@server:/path/to/
```

### 3. 启动服务

```bash
cd ev-forklift-hub
docker-compose up -d
```

### 4. 配置域名（可选）

```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    location / {
        proxy_pass http://localhost:80;
    }
}
```
