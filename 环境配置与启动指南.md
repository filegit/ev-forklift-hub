## 项目概述

新能源叉车社区平台是一个基于 Spring Cloud 微服务架构的综合性平台，包含以下核心模块：

- **efh-gateway**: API 网关 (端口: 8080)
- **efh-user**: 用户服务 (端口: 8081)
- **efh-community**: 社区服务 (端口: 8082)
- **efh-parts**: 配件服务 (端口: 8083)
- **efh-service**: 维修服务 (端口: 8084)
- **efh-common**: 公共模块

## 基础设施安装

### 1. 安装 MySQL

```bash
# Windows: 下载安装包
https://dev.mysql.com/downloads/mysql/

# 启动 MySQL 服务
net start mysql
```

### 2. 安装 Redis

```bash
# Windows: 下载安装包
https://github.com/microsoftarchive/redis/releases

# 启动 Redis
redis-server.exe
```

### 3. 安装 Nacos

```bash
# 下载 Nacos
https://github.com/alibaba/nacos/releases

# 解压后进入 bin 目录
cd nacos/bin

# Windows 单机模式启动
startup.cmd -m standalone

# 访问控制台: http://localhost:8848/nacos
# 默认账号密码: nacos/nacos
```

### 4. 安装 Kafka（可选）

```bash
# 下载 Kafka
https://kafka.apache.org/downloads

# 启动 Zookeeper
bin\windows\zookeeper-server-start.bat config\zookeeper.properties

# 启动 Kafka
bin\windows\kafka-server-start.bat config\server.properties
```

### 5. 安装 Elasticsearch（可选）

```bash
# 下载 Elasticsearch
https://www.elastic.co/downloads/elasticsearch

# 启动 Elasticsearch
bin\elasticsearch.bat

# 访问: http://localhost:9200
```

## 配置说明

### 1. Nacos 配置

所有微服务都使用 Nacos 作为配置中心，配置文件位于：
- `bootstrap.yml`: 启动配置（Nacos 连接信息）
- `application.yml`: 应用配置（数据源、Redis、Kafka 等）

**重要配置项：**

```yaml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: 127.0.0.1:8848  # Nacos 地址
        namespace: dev                # 命名空间
      config:
        server-addr: 127.0.0.1:8848  # Nacos 地址
        namespace: dev                # 命名空间
        file-extension: yml           # 配置文件格式
```

### 2. 数据库配置

修改各服务的 `application.yml` 中的数据库连接信息：

```yaml
spring:
  shardingsphere:
    datasource:
      ds0:
        jdbc-url: jdbc:mysql://localhost:3306/数据库名
        username: root      # 修改为你的用户名
        password: 123456    # 修改为你的密码
```

### 3. Redis 配置

```yaml
spring:
  redis:
    host: 127.0.0.1  # Redis 地址
    port: 6379       # Redis 端口
    database: 0      # 数据库索引
```

### 4. Kafka 配置（可选）

```yaml
spring:
  kafka:
    bootstrap-servers: 127.0.0.1:9092  # Kafka 地址
```

### 5. Elasticsearch 配置（可选）

```yaml
spring:
  elasticsearch:
    uris: http://127.0.0.1:9200  # Elasticsearch 地址
    username: elastic             # 用户名
    password: 123456              # 密码
```

## 启动步骤

### 方式一：IDEA 启动（推荐开发环境）

#### 1. 启动基础设施
确保以下服务已启动：
- ✅ MySQL
- ✅ Redis
- ✅ Nacos
- ⚠️ Kafka（可选，社区服务需要）
- ⚠️ Elasticsearch（可选，搜索功能需要）

#### 2. 编译项目

```bash
# 在项目根目录执行
mvn clean install -DskipTests
```

#### 3. 启动微服务

**按以下顺序启动：**

1. **启动网关服务**
   - 运行 `efh-gateway/src/main/java/com/efh/gateway/GatewayApplication.java`
   - 等待控制台输出：`========== Gateway服务启动成功 ==========`

2. **启动用户服务**
   - 运行 `efh-user/src/main/java/com/efh/user/UserApplication.java`
   - 等待控制台输出：`========== 用户服务启动成功 ==========`

3. **启动社区服务**
   - 运行 `efh-community/src/main/java/com/efh/community/CommunityApplication.java`
   - 等待控制台输出：`========== 社区服务启动成功 ==========`

4. **启动配件服务**
   - 运行 `efh-parts/src/main/java/com/efh/parts/PartsApplication.java`
   - 等待控制台输出：`========== 配件服务启动成功 ==========`

5. **启动维修服务**
   - 运行 `efh-service/src/main/java/com/efh/service/ServiceApplication.java`
   - 等待控制台输出：`========== 维修服务启动成功 ==========`

### 方式二：命令行启动

```bash
# 1. 编译打包
mvn clean package -DskipTests

# 2. 启动网关
cd efh-gateway/target
java -jar efh-gateway-1.0.0.jar

# 3. 启动用户服务
cd efh-user/target
java -jar efh-user-1.0.0.jar

# 4. 启动社区服务
cd efh-community/target
java -jar efh-community-1.0.0.jar

# 5. 启动配件服务
cd efh-parts/target
java -jar efh-parts-1.0.0.jar

# 6. 启动维修服务
cd efh-service/target
java -jar efh-service-1.0.0.jar
```

## 常见问题

### 1. Nacos 连接失败

**问题：** 服务启动时报错 `Connection refused: connect`

**解决方案：**
- 确认 Nacos 已启动：访问 http://localhost:8848/nacos
- 检查 `bootstrap.yml` 中的 Nacos 地址是否正确
- 检查防火墙是否阻止了 8848 端口

### 2. 数据库连接失败

**问题：** 启动时报错 `Communications link failure`

**解决方案：**
- 确认 MySQL 服务已启动
- 检查数据库名、用户名、密码是否正确
- 确认数据库已创建：`SHOW DATABASES;`
- 检查 MySQL 端口是否为 3306

### 3. Redis 连接失败

**问题：** 启动时报错 `Unable to connect to Redis`

**解决方案：**
- 确认 Redis 服务已启动：`redis-cli ping` 应返回 `PONG`
- 检查 Redis 地址和端口是否正确
- 检查 Redis 是否设置了密码

### 4. 端口被占用

**问题：** 启动时报错 `Port 8080 was already in use`

**解决方案：**
```bash
# Windows 查看端口占用
netstat -ano | findstr :8080

# 结束占用端口的进程
taskkill /PID 进程ID /F

# 或修改 application.yml 中的端口号
```

### 5. 内存不足

**问题：** 启动时报错 `OutOfMemoryError`

**解决方案：**
```bash
# 增加 JVM 堆内存
java -Xms512m -Xmx1024m -jar xxx.jar

# 或在 IDEA 中设置 VM options:
-Xms512m -Xmx1024m
```

### 6. 依赖下载失败

**问题：** Maven 构建时依赖下载失败

**解决方案：**
- 配置国内 Maven 镜像（阿里云）
- 删除本地仓库中的损坏文件：`~/.m2/repository`
- 重新执行：`mvn clean install -U`

### 7. Kafka 不可用（可选）

**问题：** 社区服务启动失败，提示 Kafka 连接失败

**解决方案：**
- 如果不需要 Kafka 功能，可以注释掉相关配置
- 或启动 Kafka 服务（需要先启动 Zookeeper）

### 8. Elasticsearch 不可用（可选）

**问题：** 社区服务启动失败，提示 Elasticsearch 连接失败

**解决方案：**
- 如果不需要搜索功能，可以注释掉相关配置
- 或启动 Elasticsearch 服务
