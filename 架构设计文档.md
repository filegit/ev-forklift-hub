# 微服务架构技术文档

## 项目概述

本项目是一个基于 **Spring Boot + Spring Cloud Alibaba** 的微服务架构系统，实现了新能源叉车社区平台的完整功能。

## 技术架构

### 核心技术栈

```
├── Spring Boot 2.7.18          # 基础框架
├── Spring Cloud 2021.0.8       # 微服务框架
├── Spring Cloud Alibaba 2021.0.5.0  # 阿里巴巴微服务组件
├── Nacos 2.2.3                 # 服务注册与配置中心
├── Spring Cloud Gateway        # API网关
├── OpenFeign                   # 服务间调用
├── MyBatis-Plus 3.5.3.1       # ORM框架
├── ShardingSphere 5.3.2       # 分库分表
├── Redis 6.x                   # 缓存
├── Redisson 3.23.4            # 分布式锁
├── Kafka 3.0.9                # 消息队列
├── Elasticsearch 7.17.9       # 搜索引擎
└── MySQL 8.0                   # 关系数据库
```

## 微服务模块

### 1. efh-gateway（API网关）
- **端口**: 8080
- **功能**: 
  - 统一入口
  - 路由转发
  - 认证鉴权
  - 限流熔断

### 2. efh-user（用户服务）
- **端口**: 8081
- **功能**:
  - 用户注册/登录
  - 用户信息管理
  - JWT认证
- **分库分表**: 2库 × 2表

### 3. efh-community（社区服务）
- **端口**: 8082
- **功能**:
  - 帖子发布/查询
  - 评论管理
  - 全文搜索（ES）
- **分库分表**: 2库 × 4表

### 4. efh-parts（配件服务）
- **端口**: 8083
- **功能**:
  - 配件管理
  - 订单处理
- **分表**: 4张表

### 5. efh-service（维修服务）
- **端口**: 8084
- **功能**:
  - 维修订单管理
  - 服务预约
- **分表**: 4张表

## 核心功能实现

### 1. 服务注册与发现（Nacos）

```yaml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: 127.0.0.1:8848
        namespace: dev
      config:
        server-addr: 127.0.0.1:8848
        namespace: dev
        file-extension: yml
```

**特点**:
- 自动服务注册
- 健康检查
- 动态配置刷新

### 2. 分库分表（ShardingSphere）

**用户表分片策略**:
```yaml
tables:
  user:
    actual-data-nodes: ds$->{0..1}.user_$->{0..1}
    database-strategy:
      sharding-column: id
      algorithm-expression: ds$->{id % 2}
    table-strategy:
      sharding-column: id
      algorithm-expression: user_$->{id % 2}
```

**分片规则**:
- 用户服务: 按用户ID分2库2表
- 社区服务: 按用户ID分2库，按帖子ID分4表
- 订单服务: 按订单ID分4表

### 3. Redis缓存

**缓存策略**:
```java
@Service
public class UserServiceImpl {
    @Autowired
    private RedisCache redisCache;
    
    public User getUserById(Long id) {
        String key = "user:info:" + id;
        User user = (User) redisCache.get(key);
        if (user == null) {
            user = userMapper.selectById(id);
            redisCache.set(key, user, 30, TimeUnit.MINUTES);
        }
        return user;
    }
}
```

**应用场景**:
- 用户信息缓存
- 热点数据缓存
- 分布式锁

### 4. 分布式锁（Redisson）

**注解方式**:
```java
@DistributedLockable(key = "user:register:#{username}", waitTime = 3, leaseTime = 10)
public String register(RegisterVO vo) {
    // 业务逻辑
}
```

**编程方式**:
```java
@Autowired
private DistributedLock distributedLock;

public void doSomething() {
    String lockKey = "business:lock:key";
    boolean locked = distributedLock.tryLock(lockKey, 3, 10, TimeUnit.SECONDS);
    try {
        // 业务逻辑
    } finally {
        distributedLock.unlock(lockKey);
    }
}
```

### 5. Kafka消息队列

**生产者**:
```java
@Autowired
private KafkaTemplate<String, String> kafkaTemplate;

public void sendMessage() {
    kafkaTemplate.send("user-register-topic", "message");
}
```

**消费者**:
```java
@KafkaListener(topics = "user-register-topic", groupId = "efh-user-group")
public void handleMessage(String message) {
    log.info("收到消息: {}", message);
}
```

**Topic列表**:
- `user-register-topic`: 用户注册消息
- `user-login-topic`: 用户登录日志
- `post-create-topic`: 帖子创建消息
- `post-like-topic`: 帖子点赞消息

### 6. Elasticsearch全文搜索

**文档定义**:
```java
@Document(indexName = "post", createIndex = true)
public class PostDocument {
    @Id
    private Long id;
    
    @Field(type = FieldType.Text, analyzer = "ik_max_word")
    private String title;
    
    @Field(type = FieldType.Text, analyzer = "ik_max_word")
    private String content;
}
```

**搜索接口**:
```java
public interface PostEsRepository extends ElasticsearchRepository<PostDocument, Long> {
    List<PostDocument> findByTitleContaining(String title);
    List<PostDocument> findByContentContaining(String content);
}
```

### 7. 并发编程

**线程池配置**:
```java
@Configuration
public class ThreadPoolConfig {
    @Bean(name = "asyncExecutor")
    public Executor asyncExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(Runtime.getRuntime().availableProcessors());
        executor.setMaxPoolSize(Runtime.getRuntime().availableProcessors() * 2);
        executor.setQueueCapacity(500);
        executor.setThreadNamePrefix("Async-");
        executor.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());
        return executor;
    }
}
```

**异步任务**:
```java
@Async("asyncExecutor")
public CompletableFuture<Void> sendMessageAsync(User user) {
    return CompletableFuture.runAsync(() -> {
        kafkaTemplate.send("topic", message);
    });
}
```

### 8. JVM优化

**G1垃圾收集器配置**:
```bash
-Xms512m                          # 初始堆大小
-Xmx1024m                         # 最大堆大小
-Xmn256m                          # 年轻代大小
-XX:MetaspaceSize=128m            # 元空间初始大小
-XX:MaxMetaspaceSize=256m         # 元空间最大大小
-XX:+UseG1GC                      # 使用G1收集器
-XX:MaxGCPauseMillis=200          # 最大GC停顿时间

```

## 技术亮点总结

✅ **微服务架构**: Spring Cloud Alibaba全家桶  
✅ **服务治理**: Nacos注册中心 + 配置中心  
✅ **API网关**: Spring Cloud Gateway统一入口  
✅ **服务调用**: OpenFeign声明式调用  
✅ **分库分表**: ShardingSphere水平扩展  
✅ **分布式缓存**: Redis + Redisson分布式锁  
✅ **消息驱动**: Kafka异步解耦  
✅ **全文搜索**: Elasticsearch + IK分词  
✅ **并发编程**: 线程池 + 异步任务 + CompletableFuture  
✅ **JVM调优**: G1GC + 内存优化 + 性能监控  
✅ **健康检查**: Actuator监控端点
