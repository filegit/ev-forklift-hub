### 4. 数据库设计 ✅

**分库策略：** 按用户 ID 取模分为 2 个库
- `efh_user_0` - 存储 ID 为偶数的用户
- `efh_user_1` - 存储 ID 为奇数的用户

**用户表结构：**
```sql
CREATE TABLE user (
  id BIGINT PRIMARY KEY,
  username VARCHAR(50) UNIQUE,
  password VARCHAR(100),
  nickname VARCHAR(50),
  phone VARCHAR(20),
  email VARCHAR(100),
  user_type INT DEFAULT 1,
  status INT DEFAULT 1,
  points INT DEFAULT 0,
  create_time DATETIME,
  update_time DATETIME
);
```

---

### 5. 缓存策略 ✅

**Redis 缓存键：**
- `user:info:id:{userId}` - 按 ID 缓存
- `user:info:username:{username}` - 按用户名缓存

**缓存过期时间：** 30 分钟

**缓存更新策略：**
- 注册时：写入缓存
- 登录时：写入缓存
- 查询时：缓存未命中则查数据库并写入缓存

## 💡 重要提示

### 1. 密码安全
- ✅ 使用 BCrypt 加密，不可逆
- ✅ 每次加密结果不同（加盐）
- ✅ 数据库中看不到明文密码

### 2. Token 安全
- ✅ JWT Token 有效期 7 天
- ✅ Token 包含用户 ID 和用户名
- ✅ Token 使用 HS256 算法签名

### 3. 缓存策略
- ✅ 缓存过期时间 30 分钟
- ✅ 按 ID 和用户名双重缓存
- ✅ 缓存失败不影响主流程

### 4. 分库策略
- ✅ 按用户 ID 取模分库
- ✅ ShardingSphere 自动路由
- ✅ 对应用层透明
