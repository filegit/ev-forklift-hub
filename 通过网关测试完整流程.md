## 🎯 完整测试脚本（Windows PowerShell）

```powershell
# 1. 注册用户
$registerResponse = Invoke-RestMethod -Uri "http://localhost:8080/user/api/register" `
  -Method Post `
  -ContentType "application/json" `
  -Body '{"username":"testuser003","password":"123456","nickname":"测试用户003","phone":"13800138003"}'

$token = $registerResponse.data.token
Write-Host "Token: $token"

# 2. 发布帖子
Invoke-RestMethod -Uri "http://localhost:8080/community/api/post" `
  -Method Post `
  -ContentType "application/json" `
  -Headers @{"Authorization"="Bearer $token"} `
  -Body '{"title":"测试帖子","content":"测试内容","category":1}'

# 3. 查看帖子列表
Invoke-RestMethod -Uri "http://localhost:8080/community/api/post/list"

# 4. 查看帖子详情
Invoke-RestMethod -Uri "http://localhost:8080/community/api/post/1"

# 5. 点赞帖子
Invoke-RestMethod -Uri "http://localhost:8080/community/api/post/1/like" `
  -Method Post `
  -Headers @{"Authorization"="Bearer $token"}
```

## 🔍 网关路由说明

### 用户服务路由
- **路径：** `/user/**`
- **目标：** efh-user (8081)
- **示例：** `http://localhost:8080/user/api/login`

### 社区服务路由
- **路径：** `/community/**`
- **目标：** efh-community (8082)
- **示例：** `http://localhost:8080/community/api/post/list`

### 配件服务路由
- **路径：** `/parts/**`
- **目标：** efh-parts (8083)

### 维修服务路由
- **路径：** `/service/**`
- **目标：** efh-service (8084)

## ✅ 网关的优势

1. **统一入口** - 所有请求通过 8080 端口
2. **统一认证** - Token 由网关统一验证
3. **负载均衡** - 自动分发请求到多个实例
4. **路由转发** - 自动转发到对应的微服务
5. **跨域处理** - 统一处理 CORS

## 🐛 常见问题

### Q: 直接访问微服务端口（8081、8082）可以吗？

**A:** 可以，但不推荐！

- ✅ **推荐：** 通过网关访问 `http://localhost:8080/user/api/login`
- ❌ **不推荐：** 直接访问 `http://localhost:8081/api/login`

**原因：**
- 网关提供统一认证
- 网关提供负载均衡
- 生产环境只暴露网关端口

---

### Q: Token 在哪里验证？

**A:** Token 由网关统一验证！

- 网关收到请求 → 验证 Token → 转发到微服务
- 微服务不需要验证 Token，直接使用网关传递的用户信息
