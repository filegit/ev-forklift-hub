# 🚀 服务器部署完整指南

## 📋 部署方案概述

本项目采用 **Docker + Docker Compose** 完整容器化部署，支持一键部署和快速更新。

---

## 🎯 部署架构

```
服务器
├── MySQL (Docker)
├── Redis (Docker)
├── Nacos (Docker)
├── 前端 Web (Docker + Nginx)
├── API 网关 (Docker)
├── 用户服务 (Docker)
├── 社区服务 (Docker)
└── 其他微服务 (Docker)
```

**优势：**
- ✅ 一键部署所有服务
- ✅ 环境隔离，互不影响
- ✅ 快速回滚
- ✅ 易于扩展

## 📦 第一步：准备服务器

### 1.1 服务器要求

**最低配置：**
- CPU: 2核
- 内存: 4GB
- 硬盘: 50GB
- 系统: Ubuntu 20.04+ / CentOS 7+

**推荐配置：**
- CPU: 4核
- 内存: 8GB
- 硬盘: 100GB

### 1.2 安装 Docker

```bash
# Ubuntu/Debian
curl -fsSL https://get.docker.com | sh
sudo systemctl start docker
sudo systemctl enable docker

# 验证安装
docker --version
```

### 1.3 安装 Docker Compose

```bash
# 下载 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

# 添加执行权限
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker-compose --version
```

### 1.4 配置防火墙

```bash
# Ubuntu
sudo ufw allow 80/tcp     # 前端
sudo ufw allow 443/tcp    # HTTPS
sudo ufw allow 8080/tcp   # 网关（可选，生产环境建议关闭）
sudo ufw enable

# CentOS
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --permanent --add-port=443/tcp
sudo firewall-cmd --reload
```

## 📤 第二步：上传项目到服务器

### 方式一：使用 Git（推荐）

```bash
# 在服务器上
cd /opt
sudo git clone https://github.com/your-repo/ev-forklift-hub.git
cd ev-forklift-hub
```

### 方式二：使用 SCP 上传

```bash
# 在本地 Windows PowerShell
# 先打包项目
cd d:\LH\ev-forklift-hub
tar -czf ev-forklift-hub.tar.gz .

# 上传到服务器
scp ev-forklift-hub.tar.gz user@your-server-ip:/opt/

# 在服务器上解压
ssh user@your-server-ip
cd /opt
tar -xzf ev-forklift-hub.tar.gz
cd ev-forklift-hub
```

### 方式三：使用 FTP 工具

使用 FileZilla 或 WinSCP 上传整个项目目录到服务器 `/opt/ev-forklift-hub`

## 🔧 第三步：配置生产环境

### 3.1 创建后端 Dockerfile

创建 `docker/Dockerfile`（用于构建后端微服务）：

```dockerfile
FROM openjdk:17-jdk-slim

# 设置工作目录
WORKDIR /app

# 复制 jar 包
ARG MODULE_NAME
COPY ${MODULE_NAME}/target/*.jar app.jar

# 暴露端口
EXPOSE 8080

# 启动应用
ENTRYPOINT ["java", "-jar", "app.jar"]
```

### 3.2 更新 docker-compose.yml

在项目根目录创建完整的 `docker-compose.yml`：

```yaml
version: '3.8'

services:
  # MySQL 数据库
  mysql:
    image: mysql:8.0
    container_name: efh-mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: your_strong_password_here  # 修改密码
      TZ: Asia/Shanghai
    ports:
      - "3306:3306"
    volumes:
      - mysql-data:/var/lib/mysql
      - ./docker/mysql/init:/docker-entrypoint-initdb.d
    command:
      - --character-set-server=utf8mb4
      - --collation-server=utf8mb4_unicode_ci
    networks:
      - efh-network
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5

  # Redis 缓存
  redis:
    image: redis:7-alpine
    container_name: efh-redis
    restart: always
    ports:
      - "6379:6379"
    command: redis-server --requirepass your_redis_password_here  # 修改密码
    volumes:
      - redis-data:/data
    networks:
      - efh-network
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  # Nacos 服务注册中心
  nacos:
    image: nacos/nacos-server:v2.2.3
    container_name: efh-nacos
    restart: always
    environment:
      MODE: standalone
      SPRING_DATASOURCE_PLATFORM: mysql
      MYSQL_SERVICE_HOST: mysql
      MYSQL_SERVICE_PORT: 3306
      MYSQL_SERVICE_DB_NAME: nacos_config
      MYSQL_SERVICE_USER: root
      MYSQL_SERVICE_PASSWORD: your_strong_password_here  # 与 MySQL 密码一致
    ports:
      - "8848:8848"
    volumes:
      - nacos-data:/home/nacos/data
    networks:
      - efh-network
    depends_on:
      mysql:
        condition: service_healthy

  # API 网关
  gateway:
    build:
      context: ./efh-gateway
      dockerfile: ../docker/Dockerfile.gateway
    container_name: efh-gateway
    restart: always
    ports:
      - "8080:8080"
    environment:
      SPRING_PROFILES_ACTIVE: docker
      NACOS_SERVER_ADDR: nacos:8848
    networks:
      - efh-network
    depends_on:
      - nacos
      - redis

  # 用户服务
  user-service:
    build:
      context: ./efh-user
      dockerfile: ../docker/Dockerfile.user
    container_name: efh-user
    restart: always
    environment:
      SPRING_PROFILES_ACTIVE: docker
      NACOS_SERVER_ADDR: nacos:8848
    networks:
      - efh-network
    depends_on:
      - nacos
      - mysql
      - redis

  # 社区服务
  community-service:
    build:
      context: ./efh-community
      dockerfile: ../docker/Dockerfile.community
    container_name: efh-community
    restart: always
    environment:
      SPRING_PROFILES_ACTIVE: docker
      NACOS_SERVER_ADDR: nacos:8848
    networks:
      - efh-network
    depends_on:
      - nacos
      - mysql
      - redis

  # 前端 Web
  web:
    build:
      context: ./efh-web
      dockerfile: Dockerfile
    container_name: efh-web
    restart: always
    ports:
      - "80:80"
    networks:
      - efh-network
    depends_on:
      - gateway

networks:
  efh-network:
    driver: bridge

volumes:
  mysql-data:
  redis-data:
  nacos-data:
```

### 3.3 创建后端服务的 Dockerfile

**efh-gateway Dockerfile** (`docker/Dockerfile.gateway`):

```dockerfile
FROM maven:3.8-openjdk-17 AS build
WORKDIR /app
COPY pom.xml .
COPY efh-common ../efh-common
COPY efh-gateway .
RUN mvn clean package -DskipTests

FROM openjdk:17-jdk-slim
WORKDIR /app
COPY --from=build /app/target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

**efh-user Dockerfile** (`docker/Dockerfile.user`):


### 3.4 更新前端 nginx.conf

修改 `efh-web/nginx.conf`，指向 Docker 网关：

```nginx
server {
    listen 80;
    server_name localhost;
    
    root /usr/share/nginx/html;
    index index.html;
    
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # API 代理到 Docker 网关
    location /api/ {
        proxy_pass http://gateway:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

## 🚀 第四步：一键部署



## 🔄 第五步：后续更新流程

### 5.1 开发新功能

在本地开发完成后：

```bash
# 本地测试
# 1. 启动本地服务测试
# 2. 确认功能正常

# 提交代码
git add .
git commit -m "feat: 新功能描述"
git push origin main
```

### 5.2 更新到服务器

**方式一：Git 拉取更新（推荐）**

```bash
# SSH 到服务器
ssh user@your-server-ip

# 进入项目目录
cd /opt/ev-forklift-hub

# 拉取最新代码
git pull origin main

# 重新构建并启动服务
docker-compose up -d --build

docker-compose logs -f
```

**方式二：只更新特定服务**

```bash
# 只更新前端
docker-compose up -d --build web

# 只更新用户服务
docker-compose up -d --build user-service

# 只更新社区服务
docker-compose up -d --build community-service
```

### 5.3 快速回滚

```bash
# 回滚到上一个版本
git reset --hard HEAD~1
docker-compose up -d --build

# 或者回滚到特定版本
git reset --hard commit-hash
docker-compose up -d --build
```

```bash
# 停止所有服务
```


## 🌐 第七步：配置域名和 HTTPS

### 7.1 配置域名

在域名服务商处添加 A 记录：

```
类型: A
主机记录: @
记录值: your-server-ip
```

### 7.2 安装 Certbot

```bash
# Ubuntu
sudo apt install certbot python3-certbot-nginx

# CentOS
sudo yum install certbot python3-certbot-nginx
```

### 7.3 获取 SSL 证书

```bash
# 停止前端容器（释放 80 端口）
docker-compose stop web

# 获取证书
sudo certbot certonly --standalone -d your-domain.com

# 重启前端容器
docker-compose start web
```

### 7.4 配置 Nginx HTTPS

更新 `efh-web/nginx.conf`：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;
    
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    
    root /usr/share/nginx/html;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    location /api/ {
        proxy_pass http://gateway:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

更新 docker-compose.yml 挂载证书：

```yaml
  web:
    volumes:
      - /etc/letsencrypt:/etc/letsencrypt:ro
    ports:
      - "80:80"
      - "443:443"
```

## 📝 第八步：自动化部署脚本

创建 `deploy.sh`：

```bash
#!/bin/bash

echo "🚀 开始部署..."

# 拉取最新代码
echo "📥 拉取最新代码..."
git pull origin main

# 停止旧服务
echo "🛑 停止旧服务..."
docker-compose down

# 重新构建并启动
echo "🔨 构建并启动服务..."
docker-compose up -d --build

# 等待服务启动
echo "⏳ 等待服务启动..."
sleep 30

# 检查服务状态
echo "✅ 检查服务状态..."
docker-compose ps

# 查看日志
echo "📋 查看日志..."
docker-compose logs --tail=50

echo "🎉 部署完成！"
```

使用脚本：

```bash
chmod +x deploy.sh
./deploy.sh
```
