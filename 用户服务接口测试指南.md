## 📝 接口测试

### 方式一：使用 Postman（推荐）

#### 1. 测试接口
**GET** `http://localhost:8081/api/test`

**预期响应：**
```json
{
  "code": 200,
  "message": "success",
  "data": "用户服务运行正常！"
}
```

---

#### 2. 用户注册
**POST** `http://localhost:8081/api/register`

**请求头：**
```
Content-Type: application/json
```

**请求体：**
```json
{
  "username": "testuser001",
  "password": "123456",
  "nickname": "测试用户001",
  "phone": "13800138001"
}
```

**预期响应：**
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9.eyJ1c2VySWQiOjEsInVzZXJuYW1lIjoidGVzdHVzZXIwMDEiLCJpYXQiOjE3MDk4ODg4ODgsImV4cCI6MTcxMDQ5MzY4OH0.xxx"
  }
}
```

**保存这个 Token！** 后续接口需要用到。

---

#### 3. 用户登录
**POST** `http://localhost:8081/api/login`

**请求头：**
```
Content-Type: application/json
```

**请求体：**
```json
{
  "username": "testuser001",
  "password": "123456"
}
```

**预期响应：**
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9..."
  }
}
```

---

#### 4. 获取用户信息
**GET** `http://localhost:8081/api/info`

**请求头：**
```
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
```
（使用注册或登录返回的 Token）

**预期响应：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 5,
    "username": "testuser001",
    "nickname": "测试用户001",
    "phone": "13800138001",
    "status": 1,
    "userType": 1,
    "points": 0,
    "createTime": "2024-03-09T12:00:00",
    "updateTime": "2024-03-09T12:00:00"
  }
}
```

**注意：** 密码字段不会返回！

### 方式二：使用 curl 命令

#### 1. 测试接口
```bash
curl http://localhost:8081/api/test
```

#### 2. 用户注册
```bash
curl -X POST http://localhost:8081/api/register \
  -H "Content-Type: application/json" \
  -d "{\"username\":\"testuser002\",\"password\":\"123456\",\"nickname\":\"测试用户002\",\"phone\":\"13800138002\"}"
```

#### 3. 用户登录
```bash
curl -X POST http://localhost:8081/api/login \
  -H "Content-Type: application/json" \
  -d "{\"username\":\"testuser002\",\"password\":\"123456\"}"
```

#### 4. 获取用户信息
```bash
curl http://localhost:8081/api/info \
  -H "Authorization: Bearer YOUR_TOKEN_HERE"
```

## 🔍 验证数据

### 1. 查看数据库

**使用 Navicat 或命令行：**

```sql
-- 连接到 Docker MySQL
-- 主机: localhost:3306
-- 用户名: root
-- 密码: 123456

-- 查看用户数据（根据 ID 分库）
USE efh_user_0;
SELECT * FROM user WHERE id % 2 = 0;

USE efh_user_1;
SELECT * FROM user WHERE id % 2 = 1;
```

**应该看到：**
- 用户名、昵称、手机号
- 密码是加密的（BCrypt 格式：`$2a$10$...`）
- 状态为 1（正常）

---

### 2. 查看 Redis 缓存

```bash
# 进入 Redis 容器
docker exec -it efh-redis redis-cli -a 123456

# 查看所有用户缓存
keys user:info:*

# 查看具体的缓存内容
get user:info:id:5
get user:info:username:testuser001
```

### 场景 2：异常情况测试

#### 2.1 重复注册
```json
// 第一次注册
POST /api/register
{
  "username": "duplicate",
  "password": "123456",
  "nickname": "重复用户"
}
// 响应：200 注册成功

// 第二次注册（相同用户名）
POST /api/register
{
  "username": "duplicate",
  "password": "123456",
  "nickname": "重复用户"
}
// 预期响应：400 用户名已存在
```

#### 2.2 密码错误
```json
POST /api/login
{
  "username": "testuser001",
  "password": "wrong_password"
}
// 预期响应：400 用户名或密码错误
```

#### 2.3 用户不存在
```json
POST /api/login
{
  "username": "nonexistent",
  "password": "123456"
}
// 预期响应：400 用户名或密码错误
```

#### 2.4 Token 无效
```bash
GET /api/info
Authorization: Bearer invalid_token
# 预期响应：401 Unauthorized
```

### 场景 4：分库验证

**注册多个用户，验证分库：**

```bash
# 注册用户 1（ID 为奇数，应该在 efh_user_1）
curl -X POST http://localhost:8081/api/register \
  -H "Content-Type: application/json" \
  -d "{\"username\":\"user1\",\"password\":\"123456\",\"nickname\":\"用户1\"}"

# 注册用户 2（ID 为偶数，应该在 efh_user_0）
curl -X POST http://localhost:8081/api/register \
  -H "Content-Type: application/json" \
  -d "{\"username\":\"user2\",\"password\":\"123456\",\"nickname\":\"用户2\"}"
```

**查看数据库：**
```sql
-- 查看 efh_user_0
USE efh_user_0;
SELECT id, username FROM user;

-- 查看 efh_user_1
USE efh_user_1;
SELECT id, username FROM user;
```

**应该看到：**
- 偶数 ID 的用户在 efh_user_0
- 奇数 ID 的用户在 efh_user_1

## 🐛 常见问题

### 1. 服务启动失败

**检查：**
- Docker 服务是否运行
- MySQL 是否启动
- Redis 是否启动
- Nacos 是否启动

### 2. 接口返回 404

**检查：**
- 服务是否启动
- 端口是否正确（8081）
- 路径是否正确（/api/register）

### 3. 接口返回 500

**查看日志：**
- IDEA 控制台的错误日志
- 可能是数据库连接失败
- 可能是 Redis 连接失败

### 4. Token 验证失败

**检查：**
- Token 是否正确复制
- Authorization 头格式是否正确
- Token 是否过期（7天有效期）
