## 🚀 启动步骤

### 第一步：启动 Nacos 2.2.3

1. **下载 Nacos 2.2.3**
   ```
   https://github.com/alibaba/nacos/releases/download/2.2.3/nacos-server-2.2.3.zip
   ```

2. **启动 Nacos**
   ```bash
   cd nacos/bin
   startup.cmd -m standalone
   ```

3. **访问控制台**
   ```
   http://localhost:8848/nacos
   账号密码: nacos/nacos
   ```

4. **创建命名空间**
   - 点击"命名空间" > "新建命名空间"
   - 命名空间ID: `dev`
   - 命名空间名: `开发环境`

### 第二步：启动基础设施

- ✅ MySQL (3306)
- ✅ Redis (6379)

### 第三步：启动微服务

按顺序在 IDEA 中启动：

1. **GatewayApplication** (8080)
2. **UserApplication** (8081)
3. **CommunityApplication** (8082)
4. **PartsApplication** (8083)
5. **ServiceApplication** (8084)

### 第四步：验证

1. **查看 Nacos 控制台**
   - 切换到 `dev` 命名空间
   - 点击"服务管理" > "服务列表"
   - 应该看到所有 5 个服务

2. **测试健康检查**
   ```bash
   curl http://localhost:8080/actuator/health
   curl http://localhost:8081/actuator/health
   ```

## 📝 配置说明

### Nacos 配置中心的作用

1. **集中管理配置**
   - 所有配置文件可以在 Nacos 控制台统一管理
   - 无需重启服务即可动态刷新配置

2. **环境隔离**
   - 通过命名空间区分 dev、test、prod 环境
   - 不同环境使用不同的配置

3. **配置共享**
   - 多个服务可以共享公共配置
   - 减少配置冗余

### 当前配置模式

目前使用的是**本地配置 + Nacos 服务发现**模式：
- 配置文件：使用本地 `application.yml`
- 服务注册：注册到 Nacos
- 服务发现：从 Nacos 获取服务列表

如果需要使用 Nacos 配置中心管理配置，需要在 Nacos 控制台添加配置文件。

## ⚠️ 注意事项

### 1. 确保 Nacos 先启动

微服务启动前，必须确保 Nacos 已经完全启动。

### 2. 命名空间必须存在

如果使用 `dev` 命名空间，必须先在 Nacos 控制台创建。

### 3. 超时时间

已设置 10 秒超时时间，如果网络较慢可以适当增加：
```yaml
spring:
  cloud:
    nacos:
      config:
        timeout: 15000  # 15秒
```

### 4. 配置刷新

使用 `@RefreshScope` 注解可以实现配置动态刷新：
```java
@RefreshScope
@RestController
public class ConfigController {
    @Value("${config.value}")
    private String configValue;
}
```

## 🐛 常见问题

### 1. 连接 Nacos 超时

**解决方案**:
- 确认 Nacos 已启动
- 检查端口 8848 是否可访问
- 增加超时时间

### 2. 命名空间不存在

**解决方案**:
- 在 Nacos 控制台创建 `dev` 命名空间
- 或使用 public 命名空间（留空）

### 3. 服务注册失败

**解决方案**:
- 检查 Nacos 日志
- 确认网络连接正常
- 查看微服务启动日志
